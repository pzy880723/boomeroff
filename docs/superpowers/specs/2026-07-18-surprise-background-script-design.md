# 惊喜一下后台脚本与素材详情稳定化设计

## 目标

让“BOOMER 帮你拍一条”从点击入口开始就形成可恢复的后台脚本任务；脚本必须由五个逻辑递进镜头组成，每镜都有合理对白且字幕逐字对应对白。视频生成后，广告文案固定一次，原生 App 下载直接保存到系统相册。

## 状态与生命周期

复用 `video_generation_jobs`，不新增业务表。

- `planning`：后台正在挑素材和生成脚本。
- `script_ready`：脚本已完成，可查看和修改。
- `queued` 及之后：用户已点击生成视频，脚本已锁定并被消费。
- `failed`：脚本任务失败，再次进入可创建新任务。

`meta.flow='surprise'` 标识惊喜一下，`meta.consumed=false` 标识尚未进入视频生成。同一用户、同一门店优先恢复最近的 `planning/script_ready` 任务。关闭弹窗不会取消任务；点击换创意会放弃当前草稿并创建新任务；点击生成视频会在同一 job 上创建 shots 并启动 Director Pipeline。

## 脚本质量规则

- 固定五段，每段 3 秒，依次是钩子、进店发现、商品细节、价值体验、行动召唤。
- 每段必须同时包含真实场景、人物动作、完整对白和字幕。
- 每段对白 18-21 个汉字；全片对白 90-100 个汉字。
- 五段对白用中文逗号连接后必须逐字等于 `continuous_dialogue`。
- 字幕直接等于对应镜头最终对白，禁止另写一套摘要字幕。
- 任意两段不得出现相同对白，也不得复用 6 个以上连续汉字。
- 对白不足、重复、逻辑字段缺失或字幕不一致时，整份 JSON 退回 DeepSeek 重写；最多两次修复，仍失败则返回明确错误，不使用固定台词拼补。

## 广告文案

Director Pipeline 生成的 `meta.publish_copy` 是成片的权威文案。素材详情优先读取已有 `meta.video_copy`，其次把 `meta.publish_copy` 规范化为唯一展示格式。打开详情不调用模型，不自动变化。只有素材没有任何文案时才显示一次性“生成文案”按钮。

## 下载

原生 App 优先获得长期 Storage 的签名 URL，再由 Capacitor Filesystem 流式下载到 Cache，最后通过 Media 插件写入相册并删除缓存。视频不经过 JS Blob/Base64。保存成功后复制同一份固定广告文案。Web 端保留浏览器下载。iOS 首次系统相册权限提示不可绕过，授权后不再增加产品确认框。

## 验收

1. 打开惊喜一下后立即出现脚本生成任务，关闭再打开恢复同一 job。
2. 脚本完成后可修改，刷新或关闭再进入仍是修改后的内容。
3. 点击生成视频后该脚本被消费，成片完成后再次进入创建新脚本。
4. 短对白、重复对白、字幕不一致均无法进入视频生成。
5. 同一素材反复打开时广告文案不变。
6. 原生 App 点击下载后直接保存相册，并复制固定文案。
