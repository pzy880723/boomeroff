# 营销中心 · 最终设计(v3)

## 一、入口与结构
「我的」菜单新增「**营销中心**」(Megaphone 图标) → `/me/marketing`,内含 3 个独立工具卡片 + 素材库:

```
营销中心
├─ 顶部:本月发布概览
└─ 🖼️ 图片优化    ✍️ AI 文案    🎬 AI 视频
└─ 📚 素材库
```

三个工具**独立成页**,产物互通(图→文/图→视频),但不强行串成流水线。

---

## 二、🖼️ 图片优化(去掉所有"风格预设")

**只做修复,不做风格化。**让店员随手拍的照片回到"正常质感",不堆滤镜。

工作台:
- 上传 1–6 张原图,左侧缩略图
- 右侧大图 + **修复开关**(默认全开):
  - ☑ 自动曝光与白平衡
  - ☑ 去畸变 / 扶正
  - ☑ 降噪 + 微锐化(保留材质)
  - ☑ 去杂物(手、抹布、价签、空袋等明显干扰物)
  - ☐ 背景净化(默认关,轻度模糊杂乱背景,**不替换**)
- 可选语义微调:"光再暖一点" / "压一下右下角反光"
- 左右对比拖动 + 下载 / 再修一版 / → 带去写文案 / → 带去做视频

**明确不做**:风格预设、换背景、加贴纸/水印、美颜、改商品本体。

技术:`google/gemini-3.1-flash-image-preview`(Nano Banana 2,擅长保留主体的轻量编辑),流式预览,prompt 写死成"修复型"指令,**不暴露风格描述给模型**;30 张/人/天。

---

## 三、✍️ AI 文案

1. 上传 1–9 张图(可从图片优化直接带过来)
2. 选平台(小红书/抖音/视频号/朋友圈)+ 口吻(种草/探店/藏家分享/上新)
3. 选填:商品名 / 价格 / 卖点
4. 生成 3 条 → 标题 + 正文 + 话题标签 + 首评建议,可逐条编辑/复制/收藏

约束:100% 简体中文;BOOMER 保留英文;禁"主播/直播间/保真/保证升值";不编造价格/年代/材质。

## 四、🎬 AI 视频(关键改造)

### 改进点 1:品牌身份**预设在 Agent 系统提示里**

店员**不需要**每次告诉 AI"我们是干嘛的"。所有视频脚本生成都自动注入品牌上下文:

```
BOOMER·OFF Vintage · 国内首家标准化中古连锁品牌
- 模式:平价、海量、标准化中古杂货铺;30,000+ SKU 高密度陈列
- 时代覆盖:50s–千禧年
- 核心调性:虽古但新 · 信任可见;创造情绪体验,而非推销
- 顾客价值:稀缺性、沉浸感、治愈、翻筐乐
- 文案禁区:不说"保真/升值/直播间/主播";不编造价格/年代/材质
- 称呼:用"你/您""店员",绝不用"主播"
```

**实现**:edge function `generate-marketing-video-script` 内置 `BRAND_SYSTEM_PROMPT` 常量(从 `.lovable/memory/` 同步),并在请求时拼接到 messages 最前。同时把店铺商品类目(从 `products` 表抽 top 类目)、当前门店名(来自登录用户的 shop) 作为运行时变量注入。

→ 店员在 UI 上只需选**视频类型 + 时长 + 画幅 + 可选卖点**,不再填"店铺介绍"。

### 改进点 2:**素材充分性分析**(生成脚本前先把关)

视频类型对**素材构成**有硬性要求。每个类型预设一份"所需镜位清单":

| 类型 | 必备镜位 | 推荐镜位 |
|---|---|---|
| 探店 | 门头 / 店内全景 / 货架陈列 / 翻筐区特写 | 店员侧脸 / 顾客背影 / 灯光氛围 |
| 产品展示 | 商品正面 / 商品细节 / 商品场景 | 上手把玩 / 与同类对比 |
| 店铺氛围 | 店内全景 / 局部陈列 / 灯光特写 | 街景门头 / 顾客剪影 |
| 新品上架 | 商品多角度(≥3) / 商品细节 | 包装/标签 / 陈列位 |

**流程**:
1. 店员上传 1–10 张图 + 选类型
2. 点「✍️ 生成脚本」→ **先调用 `analyze-marketing-assets` edge fn**(多模态读图)对每张图标签化:
   - 镜位:门头 / 全景 / 货架 / 商品 / 细节 / 人物 / 街景 / 氛围…
   - 主体:店铺 / 商品 / 人 / 环境
   - 质量:可用 / 模糊 / 严重欠曝
3. 与所选类型的"必备镜位"比对 → 三种结果:

   **(a) 充足** → 直接进入脚本生成
   
   **(b) 不足但可生成(部分必备镜位缺失,可用推荐镜位顶)** → 弹"建议补拍"卡片:
   ```
   ⚠️ 这组素材做「探店视频」还差点:
   ✓ 已有:货架陈列(2)、商品特写(3)
   ✗ 缺少:门头照、店内全景
   ✗ 推荐补:翻筐区特写
   
   [📷 我去补拍再来]   [继续生成(质量可能打折)]
   ```
   
   **(c) 严重不足(必备镜位缺一半以上)** → 阻断 + 拍摄清单:
   ```
   ❌ 暂时还不能做这个类型的视频
   建议先按下面清单拍 3–5 张再回来:
   □ 店门头(白天)        □ 店内一张能看到整面货架的全景
   □ 翻筐区俯拍          □ 一处灯光氛围特写
   [📋 复制清单给同事]   [换个视频类型]
   ```

### 改进点 3:脚本生成 → **逐镜确认**才渲染

通过充分性检查后,生成结构化脚本卡片:

```
┌─ 钩子(0–2s) ──────────────────────────┐
│ 字幕:90 年代的玩具,今天还在响  [改] │
│ 画面:图 1(门头·慢推近)        [换图]│
├─ 主体(2–12s) ─────────────────────────┤
│ 镜头 1 │ 图 2 · 左滑    3s │ 字幕[改]│
│ 镜头 2 │ 图 3 · 缩放    3s │ 字幕[改]│
│ 镜头 3 │ 图 4 · 平移    4s │ 字幕[改]│
├─ 收尾(12–15s) ────────────────────────┤
│ 字幕:店内现货 · BOOMER·OFF      [改]│
│ 画面:图 5 · 定格                [换图]│
├─ BGM:lo-fi 轻松                [换]│
└─ 总时长 15s                         ┘
```

每行可改字幕 / 换图 / 调时长 / 删加镜头 / 调 BGM。两个按钮:「🔄 重新生成脚本」「✅ 确认渲染」。

**只有确认后才花渲染钱**。失败可基于同脚本「重试渲染」,无需重写脚本。

### 渲染
MVP:`render-marketing-video` edge fn,ffmpeg 拼接图片 + Ken Burns + 字幕烧录 + BGM;异步任务表 `marketing_video_jobs`;完成后下载 mp4 + 「带文案一起发」。

## 七、为什么这么设计

| 你的痛点 | 解法 |
|---|---|
| 每次都要告诉 AI 我们是谁 | 品牌上下文**预设在系统提示和共享常量里**,店员不需要重复输入 |
| 素材不够也强行生成废片 | **先分析后生成**:不足时给出具体补拍清单,严重不足时直接阻断 |
| 视频出来不能改只能重跑 | **逐镜可编辑的脚本卡片**,确认后才渲染 |
| 滤镜预设没用 | 删光预设,只做修复型编辑 |
