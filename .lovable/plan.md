# 调整 AI 润色活动描述的生成风格

## 背景
目前 `polish-activity-description` edge function 生成的文案太像"通稿/官方公告"，不符合中古杂货铺面向年轻人的调性，而且所有内容挤在一段里，不好读。

## 调整目标
把润色后的文案改成两个清晰的板块、每条要点单独成行，语气更像"店里小伙伴在跟你讲这件事"。

### 新的输出结构
固定两段、每段内部每一点都换行（用 `\n`）：

```
【活动内容】
- xxx
- xxx

【活动规则】
- xxx
- xxx
```

- 标题文字（"活动内容""活动规则"）可微调，但必须是两个板块、之间空一行。
- 每条要点一行，短句，能不写废话就不写。
- 如果草稿里没有规则相关内容，"活动规则"下就放最基础的 1-2 条（比如"活动时间以门店公告为准""最终解释权归门店所有"），不要凭空编优惠/赠品。

### 语气
- 面向年轻人，轻松、像朋友说话，可以用"咱们""来玩""顺手"这种词。
- 不要"诚邀""莅临""惊喜钜献""不容错过"之类的通稿词。
- 不用 emoji、不用 markdown 加粗、不用 #。
- 称呼用"你"，不要"您"、不要"主播/粉丝"。
- 总长度放宽到 80–260 字（含换行），让分行后不至于被压缩成一段。

## 改动范围
只改一个文件：

### `supabase/functions/polish-activity-description/index.ts`
- 重写 `systemPrompt`：明确两段结构、每点换行、年轻口语化、禁用通稿词、不要 emoji/markdown、不要编造优惠。
- `userPrompt` 末尾提示按"【活动内容】… / 【活动规则】…"两段返回，每条单独一行。
- 字数上限从 200 放宽到 260（草稿上限 300 不变）。
- 其它逻辑（鉴权、模型 `google/gemini-2.5-flash`、429/402 错误处理、返回 `{ polished }`）全部不动。

## 不改的部分
- 前端 `ActivityEditDialog.tsx`：按钮、loading、toast、textarea 都保持现状（`rows={3}` 已经能显示多行换行结果，用户可继续手动微调）。
- 数据库、活动表结构、其他 edge function、BOOMER 浮窗等无关模块都不动。
