结论先说清楚：这不是你操作问题，是当前链路设计有问题。

我查到最近那条 30 秒视频：
- 它确实走了 3 段渲染。
- 每段有原素材参考图，但 `storyboard_url` 全是空的。
- 也就是说：你前面看到/编辑的“分镜头画面”，最后没有真正作为强参考传给 Seedance。
- 现在的 `reference_image` 更像“参考素材池”，不是“照着这张分镜动起来”。所以模型会重新想象场景，导致成片和分镜不一致。

修复计划：

1. 渲染前强制做“分镜锁定检查”
   - 点击生成视频时，如果脚本里没有完整的分镜静帧，不允许直接渲染。
   - 系统先自动补齐每个镜头的静帧。
   - 如果补齐失败，前端明确显示“第几张分镜未生成，不能继续”，而不是悄悄回退到原素材图。

2. 改成“分镜静帧优先”的提交逻辑
   - 每一段提交给 Seedance 时，优先使用该段对应的 `storyboard_url`。
   - 原素材图只作为辅助参考，不能盖过分镜静帧。
   - 角色图仍保留，但不再让角色图/原图把分镜画面挤掉。

3. 分段和分镜重新绑定
   - 30 秒仍然可以是 3 段，但每段必须明确包含哪些分镜。
   - 每段的 prompt 里写清楚：“本段必须严格参考以下分镜静帧，不允许重构场景”。
   - 前端详情里显示每段实际使用了哪几张分镜图，避免黑箱。

4. 禁止静默降级成“纯原图参考”
   - 现在的问题之一是：分镜图没带上时，系统还会继续生成视频。
   - 修复后，除非你手动点“忽略分镜继续生成”，否则不会跳过分镜。
   - 如果因为真人审核导致分镜图被拒绝，会直接中文提示，并给“软通过重试 / 换参考图 / 纯文本兜底”选项。

5. 修复历史/当前草稿的状态错位
   - 生成分镜静帧后，立即把带 `storyboard_url` 的最新脚本作为唯一可渲染脚本。
   - 防止 React 状态还没更新完，用户一点击生成视频，提交的还是旧脚本。

6. 渲染详情页加“实际使用分镜”排查信息
   - 每段显示：使用了几张分镜静帧、几张原素材参考图、是否发生降级。
   - 如果一段没有分镜静帧，会直接标红：“这一段未锁分镜”。

技术改动范围：
- `MarketingVideo.tsx`
  - 渲染前自动补齐并等待分镜静帧完成。
  - 使用返回的新脚本提交，不再依赖异步 state。
  - 没有完整分镜时阻止渲染。
- `render-marketing-video`
  - 每段 payload 增加 storyboard-first 逻辑。
  - 记录 `storyboard_refs`、`raw_refs`、`reference_count` 到任务脚本里，便于详情展示。
- `seedance-submit.ts`
  - 降级时保留日志，明确告诉前端到底是不是丢掉了分镜参考。
- `VideoJobDetailPanel.tsx`
  - 显示每段是否使用分镜静帧、是否软通过、是否丢参考图。

验收标准：
- 新生成 30 秒视频时，每段任务里必须能看到 `storyboard_url`。
- 如果分镜静帧缺失，不能直接生成视频。
- 视频详情里能看到“本段使用了 X 张分镜静帧”。
- 不再出现“前端有分镜，最终视频却完全自己想象”的情况。
