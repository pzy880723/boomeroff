## 问题定位

刚刚已确认数据库本身**没问题**：
- 今天 (2026-05-18) 梦梦在 **中信泰富 A 班**
- 明天 (2026-05-19) 起梦梦才到 **728 A 班**

小精灵答错的真正原因在 `supabase/functions/spirit-chat/index.ts`：

1. **只装载"今日+明日"两天排班** —— 用户问"什么时候在 728"这种跨天问题，AI 看不到未来日期，只能瞎猜。
2. **强制按"我的门店" `myShop` 过滤** —— 第 165、168、173 行把别的门店全部砍掉。如果你当前归属是中信泰富，那 728 的排班对小精灵就**完全不存在**，于是它把"明天 728"误读成"今天 728"。
3. **班表里没带门店名字**，只剩一个班次代号，AI 没法区分"在哪家店"。
4. System Prompt 没有"日期必须严格引用上下文里的真实日期"这条铁律，模型容易凭印象答"今天"。

## 修复方案（只动 `spirit-chat` 这一个 edge function）

### 1. 扩大时间窗

把 `todayScheduleRes` / `tomorrowScheduleRes` 两次查询，合并替换成一次**未来 14 天**的查询：

```ts
.from('shift_schedules')
.select('user_id, shift_code, work_date, shop_id')
.gte('work_date', today)
.lte('work_date', addDays(today, 13))
.order('work_date');
```

### 2. 去掉门店过滤，但把店名带上

- 一次性把 `shops` 表全拉下来，建 `shopMap: id → name`。
- `describeDay` 里**不再** `if (myShop && s.shop_id !== myShop) continue;`，跨店排班全部展示，每行后面括号注明 `(门店名)`。
- "我自己今天班次"仍优先取我所在门店的那一行，避免歧义。

### 3. 重写上下文输出格式

把"今日/明日两段大块文字"改成一张紧凑的多日表，更适合 AI 引用：

```
【未来 14 天排班】(今日 = 2026-05-18)
2026-05-18(今) 中信泰富: A 梦梦/小王  B 阿强  休 你
2026-05-19(明) 中信泰富: A 小王    728总部: A 梦梦/你
...
```

同时单独保留两行更醒目的：
```
【今日 我的班次】休（中信泰富）
【明日 我的班次】A 班 09:30–18:30（728总部），同班：梦梦
```

### 4. 收紧 System Prompt 铁律

新增两条：
- **"日期/排班必须严格引用【未来 14 天排班】里的真实行，禁止凭印象使用"今天/明天"，必须明确写出具体日期（如 5 月 19 日）。"**
- **"如果上下文里查不到某人某店的排班，请直接说"我这边没有这条排班"，不要编。"**

### 5. （可选小优化）智能放大

如果用户问句里出现的人名在数据库里能匹配到 `staff_profiles.real_name` / `profiles.display_name`，**额外**把那个人未来 30 天的排班加进上下文，确保覆盖"什么时候在某店上班"这类长跨度问题。

## 不动的地方

- 前端 `useSpiritChat` / `SpiritChatPanel` 不改。
- 数据库结构 / RLS 不动。
- 其他 edge function 不动。

## 验证步骤

1. 重新部署 `spirit-chat`。
2. 重新问"梦梦什么时候在 728 上班？"——期望回答"5 月 19 日起在 728 A 班"。
3. 问"今天梦梦在哪家店？"——期望回答"今天 (5/18) 在中信泰富 A 班"。
4. 问"我明天和谁同班？"——期望按真实门店 + 同班同事回答。
