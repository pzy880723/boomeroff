# 自定义视频渲染模式梳理与默认行为修正

## 目标
让"自定义视频"页面里的两种渲染方式职责清晰、默认行为符合预期：

- **一次成片 (one_shot)**：只生成 9 张分镜静帧作为参考图，丢给 Seedance 一次推理出≤15s 整片，**不再逐段渲染、不再前端拼接**。
- **逐镜拼接 (per_shot)**：按分镜一段一段生成视频，再由 ffmpeg 合成完整片。
- **智能 (auto)**：默认等同于"逐镜拼接"，避免后端策略漂移导致用户拿到与分镜无关的整片。

## 改动点

### 1. 默认策略 = 逐镜拼接
`src/lib/renderStrategyPref.ts`
- `DEFAULT` 改为 `'per_shot'`。
- `auto` 选项的文案/含义改为 "智能 = 逐镜拼接（推荐）"，并在选择器里把 `auto` 视作 `per_shot` 的别名传给后端。

### 2. 渲染入口分流（`MarketingVideo.tsx` + `render-marketing-video`）
- 提交渲染前：若策略为 `auto`，前端强制改写为 `per_shot` 再传给后端，移除"后端自动判断"的不确定路径。
- `one_shot` 分支：
  - 仅锁定/生成 **9 张分镜静帧**作为 reference_image，**跳过**逐段切分逻辑（`segments` 只有 1 段 = 整片）。
  - 时长按当前总时长吸附到 Seedance 合法档（5s/10s）。
- `per_shot` 分支：保留现有"按分镜切段 → 顺序提交 → 拼接"流程。

### 3. UI 文案对齐（`RenderStrategyPicker.tsx`）
- 三个选项更新为：
  - **智能**：`默认 · 等同逐镜拼接`
  - **一次成片**：`9 张分镜参考图一次出片 · ≤15s · 最自然`
  - **逐镜拼接**：`每镜单独渲染再拼接 · 时长不限 · 最可控`
- 选中"智能"时下方提示明确："智能模式当前等同于逐镜拼接"。

### 4. 任务详情透明度
`VideoJobDetailPanel.tsx`：在头部展示 `策略：一次成片 / 逐镜拼接`，方便排查为何分镜没体现。

## 不动的地方
- 分镜静帧锁定逻辑、参考图优先级、安全降级、Seedance 时长吸附逻辑保持不变。
- "惊喜一下"探店流程不受影响（它本来就是 one_shot + 真实素材）。
