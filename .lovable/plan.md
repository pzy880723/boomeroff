## 问题

当前活动分享海报（`src/components/voucher/ActivityShareDialog.tsx`）有两个明显问题：

1. **活动描述基本看不到**：描述区被硬限制为最多 4 行（`wrapText(..., 4)`），但 AI 润色后的内容现在最长 500 字、且分 `【活动内容】/【内容要求】/【活动规则】` 三段，大头直接被截断成「…」。
2. **版式拥挤、层级不清**：标题、封面、描述、时间、二维码塞在一张固定 750×1334 画布上，没有给三段式正文留空间，也没有分区视觉。门店发出去客人看不到「要写小红书、几张图、几天内」这些关键条款，违背了我们刚加的"协议要写清楚"的方向。

## 目标

让海报真正承担"活动说明书 + 报名入口"两个角色：客人扫码前就能看清活动内容、内容要求、规则条款；同时风格继续沿用优惠券同款暖棕。

## 新版式（仍为 750×1334 竖版）

```text
┌──────────────────────────────────────┐
│ BOOMER-OFF              限量邀请     │  顶部品牌条
│                                      │
│ 为你专属准备                         │
│ <活动名称（最多 2 行，自动缩字号）>   │  大标题
│ 活动时间  2026.06.14 — 2026.07.14    │  金色一行
│ ─────────── 虚线 ───────────         │
│                                      │
│ [封面 16:9，可选，圆角，高度自适应]   │  没封面则跳过
│                                      │
│ 【活动内容】                         │  小标题(金色)
│  正文……自动换行，无行数硬上限       │
│                                      │
│ 【内容要求】                         │
│  正文……                             │
│                                      │
│ 【活动规则】                         │
│  正文……                             │
│ ─────────── 虚线 ───────────         │
│ [QR 280px]  扫码报名 / 立即领券      │  底部固定区
│             长按识别 / 截图转发       │
│             domain                   │
│  BOOMER · OFF · 中古限定礼遇         │
└──────────────────────────────────────┘
```

关键变化：

- **解析三段式正文**：识别 `【活动内容】/【内容要求】/【活动规则】` 标题，分别用金色 24px 粗小标题 + 22px 正文渲染；若描述里没有这些段标题（旧数据 / 用户未润色），整段当作单段正文显示。
- **取消 4 行截断**：描述按可用高度自适应，最多渲染到底部 QR 卡片上沿，超出部分末行加 "…"。
- **正文区高度自适应**：先固定底部 QR 卡片 + 底栏的位置，再把封面/标题/时间到 QR 卡片之间整块区域留给正文；封面高度从 320 收到 280，正文区可再多 ~120px。
- **时间上移**到标题下面、加一道虚线，让"活动时间 + 标题"成为视觉块 1，正文成为视觉块 2，QR 成为视觉块 3。
- **标题自适应字号**：单行 56px，超过 2 行自动降到 44px 再排版，避免长标题挤掉正文空间。
- **行距收紧**：正文行高 32（原 34）、段间距 18，节省空间。
- **POSTER_VERSION 升到 `v4`**：让所有活动下次打开自动重新生成新海报，旧的 v3 缓存失效。

## 代码改动范围

只改一个文件：`src/components/voucher/ActivityShareDialog.tsx`

1. `POSTER_VERSION = 'v4'`。
2. 新增 `parseSections(desc)` 工具：用正则切分 `【xxx】` 段标题，返回 `{ title, body }[]`。
3. 重排绘制顺序：顶部条 → 标题（自适应字号）→ 时间 → 虚线 → 封面（可选）→ 段落循环 → 底部 QR 卡片（位置从 `H - M - 280 - 30` 倒推）。
4. 正文绘制时用 `maxY = qrCardY - 40` 作为上限，超出截断并补 "…"。
5. 不动 `VoucherPoster`、不动 edge function、不动数据库。

## 不做的事

- 不改活动描述本身的存储或长度上限（仍是 500）。
- 不改 `ActivityEditDialog`、不改 `PublicActivity` 协议弹窗。
- 不改优惠券海报样式。
